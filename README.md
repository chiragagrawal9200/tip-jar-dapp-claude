# Creator Tip Jar Protocol

A full-stack Solana dApp built with **Anchor** (Rust) and **Next.js** (TypeScript).  
Creators register an on-chain tip jar; supporters send SOL tips; the protocol atomically splits every tip between the creator and a platform fee receiver.

No Jupiter. No payment APIs. Pure custom Anchor program.

## 2 — Generate a Fresh Program Keypair

> Skip this if you want to use the placeholder ID during local testing.

```bash
# Generate a new keypair and print the public key
solana-keygen grind --starts-with TipJar:1

# Or use any keypair file:
solana-keygen new -o target/deploy/tip_jar-keypair.json --no-bip39-passphrase
solana address -k target/deploy/tip_jar-keypair.json
```

Copy the public key output, then update **three places**:

```bash
# 1. programs/tip_jar/src/lib.rs  — declare_id!("YOUR_PROGRAM_ID")
# 2. Anchor.toml                  — tip_jar = "YOUR_PROGRAM_ID"
# 3. app/utils/program.ts         — PROGRAM_ID = new PublicKey("YOUR_PROGRAM_ID")
```

## 3 — Build the Anchor Program

```bash
anchor build
```

This compiles the Rust program and generates:
- `target/deploy/tip_jar.so`   — the BPF bytecode
- `target/idl/tip_jar.json`    — the IDL (authoritative)
- `target/types/tip_jar.ts`    — TypeScript types

**Copy the IDL** to the shared folder so the frontend can import it:

```bash
yarn copy-idl
# equivalent: cp target/idl/tip_jar.json idl/tip_jar.json
```

## 6 — Initialize the Platform (one-time, localnet)

After deploying you must run `initialize_platform` once before any tip jar can be created.  
The easiest way is a small script:

```bash
# From the project root, run with ts-node or create a quick script:
cat > scripts/init-platform.ts << 'EOF'
import * as anchor from "@coral-xyz/anchor";
import { PublicKey, SystemProgram } from "@solana/web3.js";

async function main() {
  const provider = anchor.AnchorProvider.env();
  anchor.setProvider(provider);
  const program = anchor.workspace.TipJar as anchor.Program;

  const [platformConfigPDA] = PublicKey.findProgramAddressSync(
    [Buffer.from("platform_config")],
    program.programId
  );

  const FEE_BPS = 250; // 2.5%
  const feeReceiver = provider.wallet.publicKey; // use your own wallet as fee receiver

  try {
    await program.methods
      .initializePlatform(FEE_BPS)
      .accounts({
        authority: provider.wallet.publicKey,
        feeReceiver,
        platformConfig: platformConfigPDA,
        systemProgram: SystemProgram.programId,
      })
      .rpc();
    console.log("Platform initialized. Config PDA:", platformConfigPDA.toBase58());
  } catch (e) {
    console.error(e);
  }
}
main();
EOF

ts-node scripts/init-platform.ts
```

## 9 — Full End-to-End Flow

1. `solana-test-validator --reset` — start local chain
2. `anchor build && anchor deploy` — compile + deploy program
3. `yarn copy-idl` — sync IDL to frontend
4. Run `scripts/init-platform.ts` — bootstrap platform config
5. `cd app && yarn dev` — start frontend
6. Open http://localhost:3000
7. Connect Phantom (set to localnet: http://127.0.0.1:8899)
8. Go to **Create Jar** → enter name → sign tx
9. Go to **Browse Jars** → find your jar → send a tip from a second wallet
10. Verify the transaction on the local explorer

## Protocol Design

### Instructions

| Instruction | Signer | PDA written |
|---|---|---|
| `initialize_platform` | platform authority | `["platform_config"]` |
| `create_tip_jar` | creator | `["tip_jar", creator_pubkey]` |
| `send_tip` | supporter | updates `tip_jar.total_tips_received` |
| `update_platform_fee` | platform authority | updates `platform_config.fee_bps` |

### Fee math (send_tip)

```
fee_amount    = floor(amount × fee_bps / 10_000)
creator_amount = amount - fee_amount
```

Both transfers are CPI calls to the System Program in the same transaction.

### Error codes

| Code | Name | Condition |
|---|---|---|
| 6000 | `FeeTooHigh` | `fee_bps > 1000` |
| 6001 | `InvalidName` | name empty or > 50 chars |
| 6002 | `InvalidAmount` | `amount == 0` |
| 6003 | `ArithmeticOverflow` | overflow in fee math |
| 6004 | `InvalidCreator` | creator key ≠ tip_jar.creator |
| 6005 | `InvalidFeeReceiver` | fee_receiver key ≠ config.fee_receiver |
| 6006 | `Unauthorized` | caller ≠ platform authority |

## Common Issues

**`anchor build` fails — missing Solana BPF target**
```bash
rustup target add bpfel-unknown-unknown
```

**Tests fail with `Account does not exist`**  
Make sure `solana-test-validator` is running before `anchor test --skip-local-validator`.

**Frontend shows "Platform not initialized"**  
Run `initialize_platform` once after deploying (step 6 above).

**Wallet not connecting in browser**  
In Phantom → Settings → Developer Settings → change network to custom RPC `http://127.0.0.1:8899`.

**`anchor deploy` says "insufficient funds"**  
```bash
solana airdrop 5   # localnet/devnet only
```
